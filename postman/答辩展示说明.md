## 在 Postman 客户端重跑

1. 双击 `start-postman-backend.cmd`。
2. 等待命令窗口显示 `MES backend started: http://127.0.0.1:18084/`，测试结束前不要关闭该窗口。
3. 打开 Postman，点击 `Import`。
4. 导入 `MES-Full-API.postman_collection.json`。
5. 导入 `MES-Local.postman_environment.json`。
6. 右上角选择“MES 本地答辩环境”。
7. 选中“**双星轮胎 MES - 全量接口测试**”，点击 `Run`。
8. 保持默认顺序，点击 `Run 双星轮胎 MES - 全量接口测试`。
9. 结束后检查 `Passed`、`Failed`、请求总数和平均响应时间。
10. 测试结束后回到后端命令窗口，按 `Ctrl+C` 停止服务。

你截图中的 `No response`、`Errors 1`、`All tests 0` 表示 Postman 无法连接 `http://127.0.0.1:18084`，不是测试断言失败。启动上述脚本后再运行即可。

答辩现场推荐展示已经生成的脱敏 PNG；Postman 客户端用于演示集合结构或现场重跑。客户端运行结果页面可能包含请求详情，不要展示登录响应体或环境变量值。

## 命令行重跑

```powershell
cd postman
npm.cmd install
npm.cmd run generate
npm.cmd test
```

重跑后会更新：

- `reports/defense-test-report.html`
- `reports/defense-test-summary.json`
- `reports/postman-junit.xml`

命令行只要存在失败断言、请求错误或 JUnit 失败，就会返回非零退出码。
