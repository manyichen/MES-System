# MES 项目答辩 PPT 文案整理

## Part 2：项目介绍、项目中使用的技术

### 1. 项目介绍
本项目是一个面向轮胎制造现场的 MES 系统，围绕产品生产全过程展开设计，覆盖了订单管理、生产计划、工单执行、领料配送、库存管理、生产报工、质量检验、设备维护、产品追溯以及权限管理等核心业务模块。系统的目标是将传统制造企业的生产管理流程数字化、可视化和规范化，提高现场执行效率，减少人为管理误差，增强生产数据的可追溯性和管理透明度。

项目不仅是一个普通的管理后台系统，更是一个面向制造现场实际业务场景的综合解决方案。通过将生产执行、仓储物流、质量控制和权限管理进行统一协同，系统能够形成从订单下达到成品追溯的一条完整生产闭环，满足现代制造企业对流程规范、数据一致和业务协同的要求。

### 2. 项目应用场景
本系统以轮胎制造企业为应用背景，重点解决生产现场中常见的问题，如工单执行不清晰、领料流程不透明、报工与计件难以追踪、质量问题无法快速定位、设备故障处理流程不规范等。系统通过结构化的业务流程和信息化管理方式，使企业管理者能够更清楚地了解现场生产状态，操作人员能够更便捷地完成日常任务，质量和仓储人员能够更高效地协同完成业务处理。

### 3. 项目技术介绍
本项目采用前后端分离的开发模式，前端使用 Vue 3、Vite、Pinia 和 Vue Router 构建 Web 管理界面，界面交互友好，页面结构清晰，便于后续维护和扩展。后端使用 Java 21、JAX-RS、JDBC 和 PostgreSQL 实现业务逻辑与数据存储，整体架构清晰，满足企业级系统对稳定性和可扩展性的要求。

系统按照 controller-service-dao-entity 的分层设计进行开发，明确了各层职责，降低了代码耦合，提高了系统的可维护性和可扩展性。同时，项目还引入了角色权限和数据范围控制机制，保证不同角色在系统中只能访问与其职责相匹配的数据和功能，增强了系统的安全性与业务规范性。

### 4. 项目创新点与扩展能力
除了传统 Web 端管理系统，本项目还加入了微信小程序功能，进一步提升了系统的现场适用性。通过微信小程序，用户可以在移动端更方便地查看任务信息、执行部分业务操作，提高生产现场的灵活性和使用效率。

此外，项目还引入了 AI 排产辅助决策功能，系统可以根据当前任务、产线、交期和库存情况提供排产建议，帮助用户在复杂的生产环境中做出更合理的决策。项目中还实现了产品二维码和追溯标签生成能力，通过二维码将产品与生产过程、质量信息和库存信息关联起来，实现更完整的产品追溯。

### 5. 项目整体价值
本项目不仅实现了一个功能完整的 MES 管理系统，更体现了从“传统制造管理”向“数字化制造管理”升级的思路。通过系统化的业务流程设计、现代化技术栈和多端协同能力，本项目具备较强的实际应用价值和未来扩展潜力。

## Part 3：人员分工及项目开发计划

### 1. 人员分工
本项目采用“模块分工协作、整体联调”的开发方式，成员之间分工明确，保证项目能够在有限时间内高效推进。

- 项目负责人：负责项目整体规划、需求梳理、架构设计、任务协调以及最终汇报材料整理。
- 前端与交互实现：负责 Web 管理界面的页面设计与交互逻辑，以及微信小程序相关界面的实现。
- 后端业务开发：负责核心业务模块的接口设计、业务流程实现、权限控制与数据库交互。
- 数据与测试支持：负责数据库设计、数据脚本整理、系统联调、测试验证与异常问题排查。
- 文档与演示支持：负责项目文档整理、操作手册编写、演示流程设计以及答辩材料准备。

通过明确分工，项目实现了“前后端协同开发、功能模块并行推进、整体联调保障质量”的良好协作模式。

### 2. 项目开发计划
本项目开发过程按照“需求分析—架构搭建—模块开发—联调测试—成果完善”的流程推进。

#### 第一阶段：需求分析与业务梳理
首先对 MES 系统的主要业务流程进行梳理，明确订单、计划、工单、仓储、质量、设备和追溯等模块之间的关系，确定系统功能边界和数据流转逻辑。

#### 第二阶段：项目框架搭建
完成前后端开发环境的搭建，确定技术栈、接口规范、数据库结构以及项目目录结构，为后续模块开发打下基础。

#### 第三阶段：核心模块开发
按照模块划分进行开发，重点实现生产执行、仓储物流、质量管理、设备维护、权限控制以及产品追溯等核心功能。同时推进微信小程序模块的接入和适配。

#### 第四阶段：系统联调与测试
对前端页面、后端接口、数据库交互以及移动端功能进行联调测试，确保系统业务流程可顺畅运行，避免模块之间的冲突和数据不一致问题。

#### 第五阶段：成果完善与答辩准备
结合项目文档、演示流程和系统展示效果，进一步完善界面展示、功能演示和答辩材料，确保项目可以顺利完成答辩展示。

## Part 4：项目亮点

### 1. 生产闭环管理
本项目最核心的亮点之一，是实现了一个较完整的制造生产闭环管理流程。从订单下达、生产排产、工单派发、领料申请、库存扣减、生产报工、质量检验到成品入库和追溯记录，系统能够将不同环节串联起来，形成一套连贯的业务逻辑。

### 2. 角色权限与数据边界控制
系统针对不同角色设置了明确的权限边界和数据范围控制，避免不同岗位在系统中越权操作。比如计划员、车间管理员、仓库管理员、质检员、生产操作工等角色都拥有各自的业务职责和权限范围，使系统更贴近实际企业管理场景。

### 3. AI 辅助决策
项目引入了 AI 排产辅助决策功能，为生产排产环节提供智能建议。系统可以根据任务状态、交期要求、产线负荷和库存情况，生成排产建议，帮助用户提高决策效率。同时，AI 提供的是“辅助建议”，最终仍然由人工确认，兼顾了智能化和业务安全性。

### 4. 产品二维码与追溯能力
项目中引入了产品二维码生成与追溯标签管理能力。在成品检验合格后，系统能够生成二维码和追溯文档，将产品与生产过程、质量信息和库存状态关联起来。这使得产品从生产到出库再到追溯的流程更加透明，增强了企业质量管控能力。

### 5. 微信小程序移动端支持
项目不仅具备 Web 管理端，还加入了微信小程序功能，扩展了系统的使用场景。微信小程序的加入使得现场操作人员可以通过移动端更轻松地查看任务、提交信息和执行部分业务操作，大大提升了生产现场的灵活性和响应速度。

### 6. 云数据库部署与可扩展性
项目还考虑了云端部署和数据库管理的现实需求，文档中明确提及使用云数据库进行数据清理、备份和验证，说明项目具备部署到云环境中的基础条件，也体现了系统对于实际生产环境的适配能力。

### 7. 模块化和可维护性
系统采用分层设计和模块化结构，各业务模块职责清晰，后续维护和扩展都更方便。无论是新增功能，还是调整业务流程，系统都具备较强的扩展潜力。

## Part 5：需求分析、用例图、原型设计

### 1. 需求分析
本项目的需求分析围绕制造企业的实际生产管理流程展开，重点分析了企业在生产执行、仓储物流、质量控制和权限管理等方面的核心需求。

#### 业务需求
- 订单和生产计划需要可视化管理；
- 工单必须能够被派发、接收和执行；
- 领料流程需要规范化、可追踪；
- 报工过程需要确保数据真实有效；
- 质量检验要与生产结果相互关联；
- 成品需要具备可追溯的信息链；
- 不同角色需要看到与其职责一致的模块和数据。

#### 功能需求
- 用户登录与角色权限控制；
- 订单、任务、工单和排产管理；
- 领料申请、批准、拣货、配送和收料流程；
- 生产报工与计件工资管理；
- 质量检验与质检结果判定；
- 产品追溯和二维码标签生成；
- 设备报修和维修流程管理。

#### 非功能需求
- 系统操作稳定、响应及时；
- 数据安全性和权限边界明确；
- 系统结构清晰，便于后续维护；
- 支持多终端使用，适应 Web 端和移动端场景。

### 2. 用例分析
基于系统业务场景，项目中的主要角色包括：

其中，典型用例包括：

- 生成产品二维码并完成追溯记录。

## Part 8：项目心得/收获

### 1. 代码规范与开发流程规范
在项目开发过程中，我们深刻体会到规范化开发的重要性。无论是前后端分层设计，还是接口规范、数据库设计和模块划分，都直接影响系统后续的扩展性和可维护性。通过这次项目，我们更加重视代码结构的清晰性、命名规范性和模块职责的边界明确。

### 2. 技术难点解决
本项目中，我们遇到的技术难点主要包括：

- 前后端分层结构的整合与协同；
- 权限控制与数据范围边界的设计；
- 业务流程的完整串联与状态流转管理；
- 二维码生成与追溯信息关联；
- 微信小程序与主系统数据协同；
- AI 辅助决策功能的合理接入与人工确认机制。

通过不断调试和优化，我们逐步掌握了如何把复杂业务流程拆解成清晰的系统功能模块。

### 3. 异常问题与经验总结
在开发过程中，我们也遇到过一些问题，比如接口联调中出现的数据不一致、权限边界控制不够准确、数据库清理后状态数据不完整等。这些问题促使我们进一步认识到：系统开发不仅要实现功能，更要关注数据完整性、流程正确性和业务安全性。

### 4. 团队协作与项目管理收获
这次项目让我们深刻体会到团队协作在软件开发中的重要性。通过分工明确、任务协同、问题及时沟通和文档持续更新，我们不仅完成了系统开发，也提升了合作沟通和项目推进能力。

### 5. 未来展望
未来，我们希望在现有基础上继续完善系统，例如增强微信小程序的业务覆盖范围、提升 AI 决策的智能化程度、优化数据可视化看板、增强系统部署与运维能力，使项目进一步从“课程项目”走向更接近实际生产环境的应用系统。
