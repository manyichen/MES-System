# 后端重构与接口审计

## 1. 审计范围

本轮扫描覆盖后端 Java 源码、测试、数据库迁移、部署配置、前端全部 API 调用和角色权限文档。接口是否保留按以下证据交叉判断：Vue 调用、后端内部调用、测试、正式业务流程、公开扫码入口和部署健康检查。

## 2. 已删除接口

| 删除项 | 原因 | 保留入口 |
|---|---|---|
| `/warehouse-locations/*` 共 5 个 | 与仓库聚合下的库位 CRUD 完全重复 | `/warehouses/locations/*` |
| `POST /equipment/{id}/status` | 与 PUT 语义重复 | `PUT /equipment/{id}/status` |
| `/equipment/{id}/repair-reports` 创建/列表 | 与正式报修资源重复，并绕过统一数据范围处理 | `/equipment-repair-reports` |
| `/equipment/{id}/maintenance-orders` | 可绕过“报修审核后转维修单”的状态机 | `/equipment-repair-reports/{id}/to-maintenance-order` |
| `/equipment/maintenance-orders/{id}` | 与维修工单资源职责重复 | `/maintenance-orders` |
| `/equipment/maintenance-plans`、`/equipment/{id}/maintenance-plans` | 与独立维护计划资源重复 | `/maintenance-plans` |
| `POST /production-tasks/{id}/release` | 实现固定抛异常；正式流程在制定工单时确认产线/工序并原子发布任务 | `POST /work-orders` |
| 旧看板指标 CRUD/聚合共 8 个 | 无消费者，且与角色看板、经营看板重复 | `/dashboard/my-summary`、`/dashboard/executive` |

清理后不存在相同 HTTP 方法和规范化路径的重复接口。权限规则对已删除路径保持默认拒绝。

## 3. 分层结构

- `controller`：32 个 JAX-RS Controller，统一放在模块 `controller` 包。
- `service`：业务校验、角色约束、状态流转、用例编排。
- `dao`：JDBC、SQL、结果映射和必须原子提交的事务。
- `entity`：请求/响应实体和数据库领域对象。

新增 `ArchitectureLayerTest` 强制约束：所有 `@Path` 必须位于 Controller 包；Controller 不得依赖 DAO 或执行 JDBC；所有 `*Service` 必须位于模块 `service` 包且不得直接执行 JDBC；DAO 不得依赖 JAX-RS。

本轮补充或拆出的 DAO 包括健康检查、认证会话、权限管理、系统维护、用户账号、个人资料、生产工单、数据范围、角色看板和经营看板。产品追溯链查询已归入 `ProductTraceDao`；轮胎批量追溯的连接与事务也由 `TireTraceDao` 管理。仓储领料的工单归属查询从 Controller 下沉到 DAO。

清理后 32 个 Controller 均不直接访问数据库；普通 Service 中不存在 `Db`、`Connection`、`PreparedStatement`、`ResultSet` 或 SQL 查询。

## 4. 角色边界

正式角色共 12 个：

`SYSTEM_ADMIN`、`HR_MANAGER`、`GENERAL_MANAGER`、`PMC_PLANNER`、`WORKSHOP_MANAGER`、`PRODUCTION_OPERATOR`、`WAREHOUSE_ADMIN`、`QUALITY_MANAGER`、`QUALITY_INSPECTOR`、`PROCESS_ENGINEER`、`EQUIPMENT_ADMIN`、`EQUIPMENT_MAINTAINER`。

已移除后端对 `WAREHOUSE_KEEPER` 的兼容分支，种子数据改用 `WAREHOUSE_ADMIN`。`database/mes_api_role_cleanup_v8.sql` 用于清理已部署数据库中的旧角色和废弃看板权限。

关键安全修正：

- 生产操作工只能为本人被派或已接收的工单发起、查询领料。
- 领料申请创建时允许真实缺料；库存充足性由仓库管理员在审批时校验。
- 外部采购入库只接受 `warehouse.inventory.adjust`，领料权限不能采购入库。
- 管理反馈列表、详情和关闭动作均校验关联工单的数据范围。
- 设备报修、维修派工、执行和验收继续由不同权限控制，不能通过旁路接口跳过状态机。

## 5. Vue 前端

旧的单页 HTML、15 个全局脚本和运行时 DOM 拼装已删除。Vue 应用按以下职责组织：

```text
src/api/          统一请求、令牌和错误处理
src/stores/       Pinia 登录会话和权限状态
src/config/       角色模块、表格和业务动作契约
src/components/   应用框架、数据表格、业务对话框、模块工作区
src/views/        登录、角色看板、经营看板、资料、公开追溯
```

前端所有导航和操作按钮按后端权限点显示；后端仍是最终授权边界。生产构建由 Tomcat 托管，公开追溯路由为 `/trace-public?token=...`。
