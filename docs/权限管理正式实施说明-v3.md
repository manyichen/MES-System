# MES 权限管理正式实施说明 v3

## 1. 权限模型

系统采用“用户 - 角色 - 权限点 - 数据范围”四层模型。前端根据权限隐藏模块、表单和操作按钮，后端再按 HTTP 方法和 API 路径强制校验。直接调用接口不能绕过权限；未配置权限的接口默认拒绝。

核心表：`mes_user`、`mes_role`、`mes_permission`、`mes_user_role`、`mes_role_permission`、`mes_role_data_scope`、`mes_user_line_scope`、`mes_user_warehouse_scope`、`mes_user_session`、`mes_audit_log`、`mes_notification`。

## 2. 角色最终边界

| 角色 | 能看什么 | 能处理什么 | 看板指标与待办 | 明确不能做什么 |
|---|---|---|---|---|
| `SYSTEM_ADMIN` 系统管理员 | 系统健康、用户、角色、权限、数据范围、会话、审计、同步日志 | 系统管理职责组：账号角色维护、权限变更执行、数据范围分配、系统健康维护、审计查看、会话巡检、同步异常查看和登录风险处理 | 用户数、有效会话、权限申请、锁定账号、失败登录、同步异常；处理权限申请和账号安全问题 | 不能查看明文密码；不能绕过审计；不能参与排产、报工、库存、质检、设备维修等具体生产事务 |
| `HR_MANAGER` 人事经理 | 用户资料、账号状态、部门岗位、用户角色与数据范围、全部计件工资明细 | 维护员工账号角色和数据范围，维护人事职责内信息 | 在册/启用/停用账号、用户角色调整 | 不能操作生产、质量、库存和设备流程 |
| `GENERAL_MANAGER` 总经理 | 全厂订单、生产、仓储、质量、设备、追溯、反馈；工资汇总 | 经营查看、异常下钻、管理督办和反馈关闭 | 待排产订单、执行工单、质量异常、故障设备、未闭环反馈 | 不能新增或修改业务；不能代替业务主管审批；不能看个人工资和权限配置 |
| `PMC_PLANNER` PMC 计划员 | 全部订单、任务、工单、齐套、缺料、相关质量设备和追溯 | 创建订单/任务/工单，齐套分析，发布任务，安排返工计划 | 待排产、待发布、缺料、待排返工 | 不能报工、审核报工、改库存、审核质检、处理维修和用户权限 |
| `WORKSHOP_MANAGER` 车间管理员 | 仅已分配产线的生产工单、车间报工、领料、相关设备和追溯；已分配产线工资汇总 | 在已分配产线内派发工单、审核报工、协调领料、上报故障、提交反馈 | 已分配产线的执行工单、待审报工、领料、影响生产的报修 | 不能访问未分配产线；不能最终排产、改库存、查看或审核质检、管理用户 |
| `PRODUCTION_OPERATOR` 生产操作工 | 仅本人被派/已接工单、本人领料状态、本人报工、本人计件工资、设备基础信息、本人反馈 | 接收本人工单，发起本人工单领料申请，提交/修改本人待审报工，上报故障和反馈 | 本人未完工工单、领料状态、报工、待审报工、计件记录；待办直达接单和报工 | 不能接别人的工单；不能看他人报工工资；不能派工、审批领料、改库存质量设备台账或用户 |
| `WAREHOUSE_ADMIN` 仓库管理员 | 公共物料主数据，以及已分配仓库的仓库、库位、库存、流水、领料、拣货、配送、机器人和缺料 | 在已分配仓库内维护仓储数据，审核领料，拣货配送，库存调整 | 已分配仓库的待审领料、待拣货、配送中、缺料预警 | 不能访问未分配仓库；不能排产、报工、审核质检维修、看工资和用户权限 |
| `QUALITY_MANAGER` 质量主管 | 全厂质检、检验项目、质量结果、返工、相关工单批次追溯 | 创建/分配质检，审核已提交结果，决定通过/复检/返工，管理返工 | 待审核结果、待分配任务、返工、不合格数量；待办直达审核列表 | 不能代替质检员录入提交；不能审核未提交结果；不能报工、改库存、发布工艺或管理用户 |
| `QUALITY_INSPECTOR` 质检员 | 仅本人质检任务、检验项目、审核状态、相关追溯、设备基础和本人反馈 | 录入本人检验项目，提交主管审核，上报故障和反馈 | 本人待检、已提交、已完成、相关返工 | 不能看/处理他人任务；不能审核自己；不能维护标准或修改生产库存用户数据 |
| `PROCESS_ENGINEER` 工艺工程师 | SOP、工艺路线、工艺参数、不良原因及相关生产质量设备追溯 | 维护主数据、工艺路线、SOP、参数和不良原因，分析返工 | 待发布 SOP、有效参数、不良原因、待分析返工 | 不能质量放行、报工审核、改库存维修状态或管理用户 |
| `EQUIPMENT_ADMIN` 设备管理员 | 全部设备、状态、报修、维修工单、维护计划和生产影响 | 维护设备台账，审核报修，派维修，验收结果，确认恢复 | 待核实报修、待派工、待验收、故障设备 | 不能报工、改库存、审核质检、排产或管理用户 |
| `EQUIPMENT_MAINTAINER` 设备维护员 | 设备基础、仅本人维修工单、相关故障、近期计划和本人反馈 | 执行本人维修，填写结果，提交验收 | 本人待修、待验收、近期计划、未转维修报修 | 不能给自己派工或验收自己；不能改设备台账及生产库存质量用户数据 |

## 3. 看板待办的确定规则

看板待办由“角色权限 + 业务状态 + 数据归属”共同计算，不是所有人看到同一份消息。

| 待办 | 业务条件 | 处理人 | 点击后进入 |
|---|---|---|---|
| 权限申请审批 | `apply_status = SUBMITTED` | 系统管理员 | 用户与角色 |
| 待排产订单 | `order_status = PENDING_PLAN` | PMC | 计划工单 |
| 缺料预警 | `alert_status = OPEN` | PMC、仓库管理员 | 计划/仓储 |
| 工单派发 | `work_order_status = CREATED` | 车间管理员 | 工单表 |
| 本人接单 | 派给本人且未接收 | 对应操作工 | 本人工单 |
| 报工审核 | `report_status = SUBMITTED` | 车间管理员 | 报工表 |
| 领料审核 | `request_status = CREATED` | 仓库管理员 | 领料表 |
| 本人质检 | 分配给本人且待检 | 对应质检员 | 质检表 |
| 质检结果审核 | `inspection_status = SUBMITTED` | 质量主管 | 质检审核表 |
| 故障审核 | `repair_status = REPORTED` | 设备管理员 | 报修表 |
| 维修派工 | `maintenance_status = CREATED` | 设备管理员 | 维修表 |
| 本人维修 | 分配给本人且未完成 | 对应维护员 | 本人维修表 |
| 维修验收 | `maintenance_status = FINISHED` | 设备管理员 | 维修验收表 |
| 管理督办 | `feedback_status = OPEN` | 总经理/管理层 | 管理反馈 |

每条待办都携带 `requiredPermission`。没有处理权限时，即使数据库存在记录，也不会显示可点击处理入口。

## 4. 已由后端强制执行的数据边界

| 数据 | 强制规则 |
|---|---|
| 制造工单 | 操作工列表只返回本人相关工单；接单人必须与派工指定人一致 |
| 报工 | 操作工只看、提交和修改本人待审报工；已审核报工不可修改 |
| 计件工资 | 操作工只看本人；总经理和车间只看汇总；人事和管理员可看明细 |
| 质检 | 质检员只看和处理本人任务；主管只能审核 `SUBMITTED` 结果，不能代录 |
| 维修 | 维护员只看本人维修单，只能完成分配给本人的维修单；管理员验收 |
| 管理反馈 | 操作工、质检员、维护员只看本人提交的反馈 |
| 车间管理 | 车间管理员只看和处理 `mes_user_line_scope` 明确分配产线关联的工单、报工、设备、维修、领料、工资汇总和看板待办 |
| 仓储管理 | 仓库管理员只看和处理 `mes_user_warehouse_scope` 明确分配仓库关联的仓库、库位、库存、流水、领料、拣货、配送和机器人；领料与机器人必须保存 `warehouse_id` |
| 用户角色 | 只有 `user.update_role` 可修改；修改后目标用户全部旧会话立即撤销 |
| API | 无令牌返回 401；权限不足返回 403；未配置权限默认拒绝 |

## 5. 当前明确的数据范围

项目当前是单工厂模型。本人范围和主管资源范围均已由后端强制执行：

| 角色 | 当前强制范围 | 未分配时 |
|---|---|---|
| 车间管理员 | `mes_user_line_scope` 中明确分配的产线；覆盖列表、详情、操作、关联数据、工资汇总和看板 | 空列表，详情和操作返回 403，不回退为生产模块全量 |
| 仓库管理员 | `mes_user_warehouse_scope` 中明确分配的仓库；覆盖列表、详情、操作、关联数据和看板 | 空列表，详情和操作返回 403，不回退为仓储模块全量 |
| PMC、质量主管、设备管理员 | 全厂职责范围 | 保持角色职责范围，后续可扩展工厂或区域范围表 |
| 操作工、质检员、维护员 | 本人数据 | 无本人关联数据时为空 |

系统管理员通过 `GET /api/access/users/{userId}/data-scopes` 查询分配，通过 `PUT /api/access/users/{userId}/data-scopes` 覆盖更新。前端“用户与角色”页面提供“分配产线/仓库”操作。新建仓库时，若创建人是仓库管理员，该仓库自动加入创建人的范围；其他新增资源不自动授予。

## 6. 登录安全规则

- Token 默认有效 8 小时，数据库只保存 Token 的 SHA-256 摘要。
- 连续 5 次密码错误锁定 15 分钟。
- 退出立即撤销当前会话。
- 修改角色立即撤销目标用户全部旧会话。
- 密码使用 PBKDF2-HMAC-SHA256 加盐散列，任何接口都不返回密码或散列。

- 统一鉴权：`backend/src/main/java/com/example/messystem/auth/AuthFilter.java`
- API 权限映射：`backend/src/main/java/com/example/messystem/auth/AuthorizationPolicy.java`
- 数据范围服务与统一过滤：`backend/src/main/java/com/example/messystem/security/DataScopeService.java`、`backend/src/main/java/com/example/messystem/security/DataScopeFilter.java`
- 角色化看板：`backend/src/main/java/com/example/messystem/dashboard/service/RoleDashboardService.java`
- 用户角色管理：`backend/src/main/java/com/example/messystem/access/AccessResource.java`
- 前端权限和看板：`frontend/js/auth.js`、`frontend/js/access.js`、`frontend/js/dashboard.js`
- 系统管理员前端入口拆分为“系统运维”“审计日志”“用户与权限”，不与生产业务模块混放。
