# MES 全功能端到端演练操作手册

建议按“1 次准备 + 6 轮演练”执行。这样既能跑通完整生产闭环，也能覆盖缺料、驳回、返工、维修、权限和安全异常。

## 一、演练前统一准备

### 1. 验收账号

验收环境默认密码均为 `123456`：

| 角色 | 账号 |
|---|---|
| 超级管理员 | `superadmin` |
| 系统管理员 | `admin` |
| 人事经理 | `mes_hr` |
| 总经理 | `mes_general` |
| PMC 计划员 | `mes_pmc` |
| 车间管理员 | `mes_workshop` |
| 生产操作工 | `mes_operator` |
| 仓库管理员 | `mes_warehouse` |
| 质量主管 | `mes_quality_mgr` |
| 质检员 | `mes_inspector` |
| 工艺工程师 | `mes_process` |
| 设备管理员 | `mes_equipment_mgr` |
| 设备维护员 | `mes_maintainer` |

每次切换角色必须先退出登录。不要在同一浏览器的多个标签页同时登录不同账号，因为登录令牌共用 `localStorage`。

答辩现场如需用一个账号连续展示全部功能，可全程使用 `superadmin`。其他验收账号主要用于演示角色边界和数据范围，不再是完成业务闭环的必要条件。

### 2. 主流程固定使用的数据

为了符合现有数据范围，完整流程建议固定选择：

| 数据 | 选择 |
|---|---|
| 产品 | `PRD-DS-RARE-2055516 双星 稀土金 乘用子午线轮胎` |
| 原料 | `MAT-NR-RSS3 天然橡胶 RSS3` |
| 产线 | `LINE-MIX-01 密炼一线` |
| 工艺 | `PROC-MIX-P1 密炼混炼` |
| 仓库 | `天然橡胶恒温仓` |
| 库位 | `天然橡胶恒温仓 A区1排` |
| 操作工 | `mes_operator` |
| 质检员 | `mes_inspector` |
| 维护员 | `mes_maintainer` |

必须这样选择的原因：

- `mes_workshop` 当前只被授权“密炼一线”。
- `mes_warehouse` 当前只被授权“天然橡胶恒温仓”。
- 目前只有“稀土金乘用子午线轮胎”配置了可用于齐套分析的 BOM。
- 该产品已有完整工艺路线和充足的天然橡胶库存。

### 3. 演练编号

每轮都使用唯一标记，例如：

```text

```

客户名称、批次号、反馈内容、备注都带上该标记，方便搜索和截图。

建议每轮记录：

```text

领料单号：
报工单号：
质检单号：
返工单号：
维修单号：
最终状态：
```

## 二、准备轮：主数据与基础功能

这一轮主要把基础新增按钮都用一次，但新建的数据不要用于正式主流程。

### 1. 工艺与主数据

登录 `mes_process`，进入“工艺与主数据”。

#### 新增产品

进入“产品主数据”：

```text
产品编码：E2E-PRODUCT-0716
产品名称：演练测试轮胎
产品型号：E2E-01
规格：测试规格
```

预期：新增成功，状态为启用。

进入“产品BOM”可继续为新产品新增、编辑或删除 BOM 项。若不准备为演练产品配置完整物料库存，主流程仍建议使用现有稀土金轮胎。

#### 新增工艺路线

进入“工艺路线”：

```text
适用产品：演练测试轮胎
工序编码：E2E-PROC-0716
工序名称：演练成型工序
工序顺序：1
工作中心：BUILDING
```

#### 新增产线

进入“生产产线”：

```text
产线编码：E2E-LINE-0716
产线名称：演练测试产线
产线类型：GENERAL
日产能：100
```

不要把正式跨角色流程派到这条线，因为车间管理员当前没有这条线的数据权限。

### 2. 仓储主数据

登录 `mes_warehouse`，进入“仓储物流”。

#### 新增物料

进入“物料主数据”：

```text
物料编码：E2E-MAT-0716
物料名称：演练辅助材料
物料类型：AUX
规格：测试规格
单位：kg
```

#### 新增仓库

进入“仓库”：

```text
仓库编码：E2E-WH-0716
仓库名称：演练测试仓
仓库类型：RAW
```

仓库管理员创建的仓库会自动加入自己的数据范围。

#### 新增库位

进入“库位”：

```text
所属仓库：演练测试仓

库位名称：演练测试库位
```

#### 外部采购入库

进入“库存台账”，点击“外部采购入库”：

```text

仓库：演练测试仓

采购数量：100
批次号：E2E-PURCHASE-0716
采购说明：全功能演练入库
```

预期：

- 库存台账出现 100。
- 产生采购入库流水。

## 三、第一轮：正常生产、合格质检、入库追溯闭环

### 阶段一：订单、任务、齐套和工单

#### 1. PMC 创建客户订单

登录 `mes_pmc`，进入“计划与工单 → 客户订单”，点击“新建订单”。

```text
客户名称：E2E-A-0716 正常流程客户
产品：双星 稀土金 乘用子午线轮胎
订单数量：20
交付日期：选择未来日期
优先级：2
```

预期状态：

```text
订单：PENDING_PLAN
```

#### 2. 创建生产任务

进入“生产任务”，点击“创建任务”。

```text
待排产订单：选择刚才的订单
目标产线：密炼一线
计划数量：20
```

预期：

```text
任务状态：CREATED
齐套状态：PENDING
订单状态：PLANNED
```

#### 3. 执行齐套分析

在该任务行点击“齐套分析”。

预期：

```text
任务状态：READY
齐套状态：READY
分析结果：物料齐套
```

如果出现 `product bom is required`，说明选错了产品。

#### 4. 使用 AI 排产辅助

进入“制造工单”，页面上方会出现“AI 排产辅助决策”。

```text
待分析任务：刚才的 READY 任务
分析周期：7 天
决策目标：优先满足交期，同时降低缺料和产线冲突风险
```

点击“生成工单决策建议”。

查看：

- 推荐产线
- 建议排期
- 交期判断
- 排产依据

可以点击“采用建议制定工单”，但最终必须人工确认：

```text
产线：密炼一线
工艺路线：PROC-MIX-P1 密炼混炼
计划数量：20
批次号：E2E-A-BATCH-0716
```

如果 AI 推荐了其他产线，必须手动改回“密炼一线”，否则车间管理员可能看不到。

#### 5. 创建制造工单

提交“制定工单”。

预期：

```text
生产任务：RELEASED
制造工单：CREATED
```

### 阶段二：派工、接单、领料和配送

#### 6. 车间管理员派工

退出 PMC，登录 `mes_workshop`。

进入“计划与工单 → 制造工单”，搜索批次号 `E2E-A-BATCH-0716`。

点击“派工”：

```text
生产操作工：mes_operator
```

预期：

```text
工单状态：DISPATCHED
```

#### 7. 操作工接单

登录 `mes_operator`，进入“生产报工 → 我的制造工单”。

找到对应工单，点击“接单”。

预期：

```text
工单状态：RECEIVED
接收人：mes_operator
```

#### 8. 操作工发起领料

进入“仓储物流 → 领料申请”，点击“发起领料”。

```text
当前制造工单：E2E-A 工单
目标仓库：天然橡胶恒温仓
领料物料：MAT-NR-RSS3 天然橡胶 RSS3
需求数量：53 kg
批次号：可不填
备注：E2E-A-0716 正常领料
```

53 kg 对应 20 条轮胎的当前 BOM 需求量。

预期：

```text
领料状态：CREATED
```

#### 9. 仓库接收领料申请

登录 `mes_warehouse`，进入“仓储物流 → 领料申请”。

找到该申请，依次执行：

1. 点击“接收”。

   ```text
   CREATED → RECEIVED
   ```

2. 点击“批准”。

   ```text
   RECEIVED → APPROVED
   ```

批准时系统会检查库存，并自动创建拣货任务。

#### 10. 完成拣货

进入“拣货任务”，找到新生成的 `CREATED` 任务，点击“完成拣货”。

预期：

```text
拣货任务：COMPLETED
配送任务：自动创建为 PENDING
```

#### 11. 确认配送到达

进入“配送任务”，找到新任务，点击“确认到达”。

预期：

```text
PENDING → ARRIVED
```

#### 12. 操作工确认收料

登录 `mes_operator`，进入“仓储物流 → 配送任务”。

点击“确认收料”。

预期：

```text
配送任务：RECEIVED
领料申请：COMPLETED
库存：扣减 53 kg
库存流水：生成 OUT 记录
```

只有发起该领料申请的操作工可以确认收料。

### 阶段三：生产报工和计件工资

#### 13. 操作工提交报工

进入“生产报工 → 生产报工”，点击“提交报工”。

```text
当前制造工单：E2E-A 工单
报工数量：20
合格数量：20
不良数量：0
工时：8
```

预期：

```text
报工状态：SUBMITTED
```

#### 14. 车间管理员审核报工

登录 `mes_workshop`，进入“生产报工 → 生产报工”。

找到该报工，点击“审核通过”。

预期：

```text
报工状态：APPROVED
工单 actual_qty：增加 20
工单状态：FINISHED
计件工资：自动生成 PENDING 记录
```

进入“计件工资”确认出现对应记录。

这一步也是“产线负荷与设备”完成数量不再为 0 的关键。

### 阶段四：质量检验与合格入库

#### 15. 质量主管创建质检单

登录 `mes_quality_mgr`，进入“质量管理 → 质量检验”。

点击“创建质检”：

```text
待检验报工单：刚才已审核的报工
检验类型：FINAL
```

预期：

```text
质检状态：CREATED
```

#### 16. 分配质检员

在该质检单点击“分配质检员”：

```text
质检员：mes_inspector
```

#### 17. 质检员录入检验项

登录 `mes_inspector`，进入“质量管理 → 质量检验”。

找到分配给自己的任务，点击“录入检验项”。

建议录入两项：

第一项：

```text
项目编码：VISUAL
项目名称：外观检查
标准值：无明显缺陷
实测值：符合
项目结论：PASS
```

第二项：

```text
项目编码：BALANCE
项目名称：动平衡检查
标准值：≤ 10g
实测值：6g
项目结论：PASS
```

#### 18. 提交质检审核

点击“提交审核”：

```text
检验结论：PASS
检验备注：E2E-A-0716 检验项目全部合格
```

预期：

```text
质检状态：SUBMITTED
```

#### 19. 质量主管最终判定

登录 `mes_quality_mgr`，找到该质检单，点击“质量判定”：

```text
判定结果：PASS
```

预期：

```text
质检状态：APPROVED
判定结果：PASS
质量追溯：自动生成 NORMAL 记录
成品库存：自动入库
```

### 阶段五：产品追溯和轮胎标签

#### 20. 生成轮胎二维码

使用 `mes_warehouse` 或 `superadmin`，进入“产品追溯 → 轮胎二维码”。

点击“生成轮胎标签”：

```text
已审核合格质检单：E2E-A 质检单
入库仓库：选择成品仓
入库库位：选择该成品仓库位
生成数量：2
```

预期：

- 生成 2 条轮胎序列号。
- 生成二维码和追溯文档。
- 轮胎状态为 `IN_STOCK`。

选择其中一条，点击“记录打印”：

```text
打印备注：E2E-A-0716 首次标签打印
```

预期打印次数加 1。

#### 21. 创建产品追溯链

登录 `superadmin`，进入“产品追溯 → 产品追溯链”。

点击“创建追溯记录”：

```text
客户订单：E2E-A 订单
生产任务：E2E-A 任务
制造工单：E2E-A 工单
批次号：E2E-A-BATCH-0716
状态：NORMAL
```

## 四、第二轮：缺料预警、采购补料和恢复齐套

当前数据库已经存在 `SHORTAGE/SHORTAGE` 任务和 `OPEN` 缺料预警，建议直接利用，避免创建超大订单污染经营看板。

### 1. PMC 发布缺料预警

登录 `mes_pmc`，进入“计划与工单 → 生产任务”。

筛选：

```text
任务状态：SHORTAGE
齐套状态：SHORTAGE
```

点击“发布缺料预警”。

预期：

```text
缺料预警：OPEN
接收角色：仓库管理员
```

### 2. 仓库接收预警

登录 `mes_warehouse`，进入“计划与工单 → 缺料预警”。

点击“接收预警”。

预期：

```text
OPEN → ACCEPTED
```

### 3. 外部采购补料

进入“仓储物流 → 库存台账 → 外部采购入库”。

按照缺料预警中的物料和缺口填写：

```text
物料：预警中的物料
仓库：天然橡胶恒温仓
库位：天然橡胶恒温仓 A区1排
采购数量：大于缺料数量
批次号：E2E-SHORTAGE-0716
采购说明：缺料预警补料
```

### 4. PMC 重新齐套分析

重新登录 `mes_pmc`，再次对原任务点击“齐套分析”。

预期：

```text
任务状态：READY
齐套状态：READY
相关缺料预警：自动 RESOLVED
```

之后可以继续制定工单，验证异常恢复后能够重新进入正常流程。

## 五、第三轮：驳回和非法操作场景

这些场景都建议使用独立的小订单，不要与第一轮共用记录。

### 1. 操作工退回已派工单

流程执行到：

```text
工单状态：DISPATCHED
```

登录 `mes_operator`，进入“计划与工单 → 制造工单”，点击“退回”。

```text
退回原因：E2E-B-0716 设备或人员条件不满足
```

预期：

```text
DISPATCHED → REJECTED
```

当前 UI 没有把 `REJECTED` 工单重新置为 `CREATED` 的按钮，因此该分支到此结束。

### 2. 仓库驳回领料申请

操作工创建一张新领料申请后，登录 `mes_warehouse`。

在 `CREATED` 或 `RECEIVED` 状态点击“驳回”：

```text
驳回原因：申请数量或批次信息不完整
```

预期：

```text
领料状态：REJECTED
不会生成拣货任务
不会扣减库存
```

### 3. 车间驳回报工

操作工提交报工：

```text
报工数量：5
合格数量：4
不良数量：1
```

车间管理员点击“驳回”：

```text
驳回原因：合格与不良数量需要重新核实
```

预期：

```text
报工状态：REJECTED
工单 actual_qty 不增加
不生成计件工资
```

当前前端没有“修改并重新提交已驳回报工”的按钮，因此该分支暂时停止。

### 4. 必测校验

| 操作 | 故意输入 | 预期 |
|---|---|---|
| 新建订单 | 交付日期早于今天 | 拒绝 |
| 新建订单 | 数量为 0 | 拒绝 |
| 齐套分析 | 选择新建但无 BOM 的产品 | 提示缺少 BOM |
| 制定工单 | 任务未齐套 | 拒绝 |
| 制定工单 | 选择维修/禁用产线 | 拒绝 |
| 批准领料 | 库存少于申请数量 | 拒绝，不生成拣货任务 |
| 确认收料 | 使用非申请人账号 | 拒绝 |
| 提交报工 | 合格数 + 不良数大于报工数 | 拒绝 |
| 提交报工 | 累计数量超过计划的 110% | 拒绝 |
| 质检提交 | 选择 FAIL/REWORK 但不写备注 | 拒绝 |
| 维修验收 | 使用执行维修的维护员验收 | 拒绝 |

## 六、第四轮：质量返工完整闭环

### 1. 先生成一张已审核报工

重新执行简化主流程：

```text
订单 → 任务 → 齐套 → 工单 → 派工 → 接单 → 报工 → 审核通过
```

建议数量为 10。

### 2. 创建并分配质检

质量主管创建质检单并分配给 `mes_inspector`。

### 3. 质检员录入不合格项目

```text
项目编码：SIDEWALL
项目名称：胎侧外观
标准值：无气泡、无裂纹
实测值：发现局部气泡
项目结论：FAIL
```

提交审核：

```text
检验结论：REWORK
检验备注：胎侧局部气泡，需要返工处理
```

### 4. 质量主管判定返工

质量主管点击“质量判定”：

```text
判定结果：REWORK
```

预期：

```text
质检状态：REWORK_REQUIRED
返工单：自动创建为 CREATED
质量追溯：REWORKED
```

### 5. PMC 纳入返工计划

登录 `mes_pmc`，进入“计划与工单 → 返工重排需求”。

找到对应返工单，点击“纳入生产计划”：

```text
返工数量：按实际返工数量填写
计划开始：当前时间之后
计划完成：晚于开始时间
目标产线：密炼一线
```

预期：

```text
返工单：PLANNED
返工生产任务：CREATED/PENDING
```

### 6. 质量主管派发返工

登录 `mes_quality_mgr`，进入“质量管理 → 返工单”，点击“派发返工”。

预期：

```text
PLANNED → DISPATCHED
```

### 7. 返工任务重新生产

PMC 对返工任务执行：

```text
齐套分析 → 制定工单
```

车间和操作工执行：

```text
派工 → 接单 → 报工 → 审核
```

质量部门可再创建 `REWORK` 类型质检，并按 PASS 流程完成复检。

### 8. 完成返工

复检通过后，质量主管在原返工单点击“完成返工”。

预期：

```text
DISPATCHED → FINISHED
```

如果想覆盖彻底不合格场景，另建一张质检单：

- 项目结论选择 `FAIL`
- 提交结果选择 `FAIL`
- 主管判定 `FAIL`

预期质检状态为 `REJECTED`，不会创建返工单。

## 七、第五轮：设备故障、维修、验收闭环

### 1. 新增设备

登录 `mes_equipment_mgr`，进入“设备维护 → 设备台账”。

点击“新增设备”：

```text
设备编码：E2E-EQ-0716
设备名称：演练密炼设备
设备类型：MIXER
所属产线：密炼一线
```

### 2. 新增维护计划

进入“维护计划”：

```text
维护设备：演练密炼设备
维护周期：MONTHLY
下次维护时间：未来日期
状态：ACTIVE
```

### 3. 上报设备故障

可以使用 `mes_operator` 上报，更符合职责分离。

进入“设备维护 → 设备台账”，点击“设备报修”：

```text
报修单号：可留空自动生成
报修设备：演练密炼设备
关联制造工单：可选择当前在制工单
故障级别：HIGH
故障描述：E2E-EQ-0716 温控波动并伴随异常振动
```

预期：

```text
报修状态：REPORTED
设备状态：FAULT
```

### 4. 设备管理员审核报修

登录 `mes_equipment_mgr`，进入“设备报修”，点击“审核并转维修”。

预期：

```text
报修单：REPORTED → CONVERTED
维修工单：自动创建为 CREATED
```

“补生成维修单”只用于报修已经是 `APPROVED`、但自动建维修单曾失败的补偿场景。正常流程会直接变成 `CONVERTED`，通常不会出现这个按钮。

### 5. 派发维修工单

进入“维修工单”，点击“派工”：

```text
设备维护员：mes_maintainer
```

预期：

```text
CREATED → ASSIGNED
```

### 6. 维护员提交维修结果

登录 `mes_maintainer`，进入“设备维护 → 维修工单”。

点击“提交维修结果”：

```text
维修结果：已校准温控模块、紧固传动连接并试运行正常
```

预期：

```text
ASSIGNED → FINISHED
```

### 7. 设备管理员验收

登录 `mes_equipment_mgr`，点击“验收”。

预期：

```text
维修工单：完成验收
设备状态：RUNNING
```

最后可在设备台账使用“更新状态”，依次测试：

```text
RUNNING → IDLE → RUNNING
```

## 八、第六轮：账号、权限、安全、反馈和经营看板

### 1. 个人资料

任意账号进入“个人资料”，修改：

```text
电话
邮箱
头像地址
个人简介
```

点击保存，重新进入页面验证数据仍存在。

### 2. 人事发起账号申请

登录 `mes_hr`，进入“用户与权限 → 账号申请”。

点击“发起账号申请”：

```text
登录账号：e2e_operator_0716
初始密码：123456
姓名：演练操作员
申请角色：PRODUCTION_OPERATOR
部门：生产部
申请说明：全流程演练账号
```

预期：

```text
申请状态：SUBMITTED
```

人事经理不能申请创建系统管理员。

### 3. 系统管理员审批账号

登录 `admin`，进入“账号申请”。

点击“通过并创建”。

预期：

```text
申请状态：APPROVED
用户账号自动创建
新账号可登录
```

再让人事提交第二张账号申请，系统管理员点击“拒绝”，验证拒绝原因必填且不会创建账号。

### 4. 权限变更申请

登录 `mes_hr`，进入“权限申请”。

```text
目标用户：刚创建的演练账号
目标角色：例如 QUALITY_INSPECTOR
申请原因：岗位调整权限演练
```

预期：

```text
申请状态：SUBMITTED
```

登录 `admin`：

1. 点击“审核”，选择 `APPROVED`。
2. 申请状态变为 `APPROVED`。
3. 点击“执行变更”。
4. 状态变为 `APPLIED`。
5. 目标用户旧会话被撤销。
6. 重新登录后看到新角色菜单。

再提交一张申请并选择 `REJECTED`，验证拒绝分支。

### 5. 超级管理员直接维护用户

登录 `superadmin`，进入“用户账号”。

测试：

- 新增用户
- 设置多个角色
- 设置数据范围

为测试用户设置：

```text
可管理产线：密炼一线
可管理仓库：天然橡胶恒温仓
```

然后登录该账号，确认只能看到授权资源。

进入“角色”和“权限点”页签，检查角色及 64 个权限点的只读列表。

### 6. 登录锁定与解除锁定

使用刚创建的测试账号：

1. 先正常登录一次，生成会话。
2. 退出。
3. 连续输入错误密码 5 次。
4. 账号应被锁定 15 分钟。
5. 登录 `admin` 或 `superadmin`。
6. 进入“下线锁定会话”。
7. 找到该账号，点击“解除锁定”。
8. 使用正确密码重新登录。

同时检查首页或系统数据中的失败登录数量。

### 7. 账号停用与恢复

超级管理员可以在“用户账号”中选择非超级管理员账号，执行“撤销会话”或“停用账号”。停用后的账号会出现在“删除账号记录”中，可继续点击“恢复账号”。

答辩时建议先新建一个带演练编号的临时账号，再演示停用和恢复；系统会拒绝超级管理员停用自己的当前账号，不需要直接操作数据库。

### 8. 管理反馈

当前管理反馈列表固定查询 `workOrderId=1`，所以推荐使用现有两条 `OPEN` 反馈测试关闭功能。

登录 `mes_general`：

1. 进入“管理反馈”。
2. 查看现有开放反馈。
3. 点击“关闭反馈”。
4. 预期状态变为 `CLOSED`。

新增反馈可以由有 `feedback.create` 权限的角色执行：

```text
关联制造工单：选择未完成工单
反馈类型：QUALITY / EQUIPMENT / MATERIAL
反馈内容：E2E-MGMT-0716 请责任部门在今日完成异常闭环
```

如果新反馈不在列表出现，原因是当前前端列表固定过滤工单 1，并非新增失败。

### 9. 总经理经营驾驶舱

登录 `mes_general` 或 `superadmin`。

检查两个页面：

#### 公司概况

验证：

- 订单计划量
- 合格产出
- 合格率
- 设备可用率
- 在制工单
- 经营风险
- 部门健康度
- 七日产出趋势
- 管理审计关注

#### 实时运行

验证：

- 在线产线
- 产线负荷
- 计划与完成数量
- 设备数量和故障数量
- 实时经营风险
- 全厂实时运行态势

第一轮报工审核通过后，“密炼一线”的完成数量应不再为 0；设备维修验收后设备可用率应恢复。

## 九、权限拒绝验收

建议每个角色至少验证一个允许操作和一个禁止操作。

| 角色 | 应允许 | 应拒绝或不显示 |
|---|---|---|
| `mes_pmc` | 创建订单、任务、齐套、工单 | 报工、质检判定、库存调整 |
| `mes_workshop` | 密炼一线派工、报工审核 | 其他产线、库存审批、质检审核 |
| `mes_operator` | 本人接单、领料、报工、报修 | 他人工单、审核自己的报工 |
| `mes_warehouse` | 天然橡胶恒温仓领料、库存、拣货 | 其他仓库、生产报工 |
| `mes_quality_mgr` | 创建、分配、审核质检 | 代替质检员录入结果 |
| `mes_inspector` | 本人质检任务录入和提交 | 他人任务、最终质量判定 |
| `mes_equipment_mgr` | 审核、派修、验收 | 生产报工、质检审核 |
| `mes_maintainer` | 本人维修结果提交 | 自己派工、自己验收 |
| `mes_process` | 产品、产线、工艺路线 | 排产、报工、库存修改 |
| `mes_hr` | 账号和权限申请 | 最终执行授权 |
| `admin` | 用户权限和系统维护 | 具体生产业务操作 |
| `mes_general` | 驾驶舱、查看、反馈关闭 | 一线业务新增和审批 |
| `superadmin` | 全部功能 | 仍不能查看明文密码或绕过审计 |

## 十、最终完成标准

六轮结束后，至少应得到：

- 1 条正常订单到合格入库、二维码追溯的完整链路
- 1 条缺料预警到采购补料再齐套的恢复链路
- 1 张退回工单
- 1 张驳回领料单
- 1 张驳回报工单
- 1 条质量返工闭环
- 1 条设备报修到验收闭环
- 1 张通过和 1 张拒绝的账号申请
- 1 张通过执行和 1 张拒绝的权限申请
- 1 次账号锁定和解除锁定
- 13 个账号的角色边界检查
- 公司概况和实时运行驾驶舱数据发生对应变化
