# 腾讯云轻量应用服务器部署步骤

服务器公网 IP：`119.45.196.92`

当前项目是 Web 应用，部署结构如下：

```text
浏览器 -> Nginx:80
静态页面 -> MES System/src/main/webapp
/api/* -> 反向代理到 127.0.0.1:8080
后端入口 -> com.example.messystem.MesBackendApplication
进程守护 -> Supervisor
```

公网只需要访问：

```text
http://119.45.196.92/
```

后端 `8080` 端口只给服务器本机 Nginx 访问，不建议对公网开放。

## 1. 放通公网端口

在腾讯云轻量应用服务器控制台的防火墙规则中添加：

```text
协议：TCP
端口：80
来源：0.0.0.0/0
说明：MES Web
```

后续如果配置 HTTPS，再额外开放 `443`。

## 2. 安装运行环境

以下命令适用于 Ubuntu/Debian：

```bash
sudo apt update
sudo apt install -y openjdk-21-jdk maven git nginx supervisor
java -version
mvn -version
nginx -v
supervisord -v
```

必须确认 `java -version` 是 JDK 21。

## 3. 拉取项目代码

推荐把仓库放到 `/www/wwwroot/mes`：

```bash
sudo mkdir -p /www/wwwroot /www/wwwlogs
cd /www/wwwroot
sudo git clone <你的远程仓库地址> mes
sudo chown -R root:root /www/wwwroot/mes
```

主工程目录名是 `MES System`，路径中有空格。为了让 Nginx 和 Supervisor 配置稳定，创建一个无空格软链接：

```bash
sudo ln -sfn "/www/wwwroot/mes/MES System" /www/wwwroot/mes-app
```

后续部署统一使用：

```text
/www/wwwroot/mes-app
```

## 4. 配置数据库环境变量

在服务器仓库根目录创建 `.env`：

```bash
cd /www/wwwroot/mes
sudo cp deploy/env.production.example .env
sudo nano .env
```

填写真实数据库配置：

```env
MES_DB_HOST=你的 PostgreSQL 地址
MES_DB_PORT=5432
MES_DB_NAME=MESSystem
MES_DB_USER=MESSystem
MES_DB_PASSWORD=你的数据库密码
MES_HOST=127.0.0.1
MES_PORT=8080
MES_OPEN_BROWSER=false
MES_PUBLIC_URL=http://119.45.196.92/
```

`.env` 不要提交到 Git。

## 5. 本机测试后端入口

```bash
cd /www/wwwroot/mes-app
chmod +x run-backend.sh
MES_HOST=127.0.0.1 MES_PORT=8080 MES_OPEN_BROWSER=false ./run-backend.sh
```

看到类似输出即表示后端启动成功：

```text
MES backend started: http://127.0.0.1:8080/
Listening on 127.0.0.1:8080
```

另开一个 SSH 窗口测试：

```bash
curl http://127.0.0.1:8080/api/db/ping
```

测试完成后，在运行后端的窗口按 `Ctrl+C` 停止。

## 6. 配置 Supervisor

项目提供了 Supervisor 模板：

```text
deploy/supervisor/mes-backend.conf
```

复制并启用：

```bash
sudo cp /www/wwwroot/mes/deploy/supervisor/mes-backend.conf /etc/supervisor/conf.d/mes-backend.conf
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl status
```

常用命令：

```bash
sudo supervisorctl restart mes-backend
sudo supervisorctl stop mes-backend
sudo supervisorctl start mes-backend
```

如果状态不是 `RUNNING`，查看日志：

```bash
sudo tail -n 100 /www/wwwlogs/mes-backend-error.log
sudo tail -n 100 /www/wwwlogs/mes-backend-access.log
```

## 7. 配置 Nginx

项目提供了 Nginx 模板：

```text
deploy/nginx/mes.conf
```

复制并启用：

```bash
sudo cp /www/wwwroot/mes/deploy/nginx/mes.conf /etc/nginx/sites-available/mes.conf
sudo ln -sfn /etc/nginx/sites-available/mes.conf /etc/nginx/sites-enabled/mes.conf
```

如果默认站点占用了 80 端口首页，可以移除默认站点链接：

```bash
sudo rm -f /etc/nginx/sites-enabled/default
```

检查并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 8. 验证部署结果

服务器内部验证：

```bash
curl http://127.0.0.1:8080/api/db/ping
curl http://127.0.0.1/api/db/ping
```

本机浏览器访问：

```text
http://119.45.196.92/
```

如果网页能打开，页面中的订单、工单、仓储、质量、设备、看板等接口能加载数据，就说明前后端链路成功。

## 9. 后续更新部署

每次组员推送新代码后，在服务器执行：

```bash
cd /www/wwwroot/mes
sudo git pull
sudo ln -sfn "/www/wwwroot/mes/MES System" /www/wwwroot/mes-app
cd /www/wwwroot/mes-app
sudo chmod +x run-backend.sh
sudo supervisorctl restart mes-backend
sudo nginx -t
sudo systemctl reload nginx
```

如果只改了前端静态页面，通常重载 Nginx 即可；如果改了 Java 后端代码，要重启 `mes-backend`。

## 10. 当前保留的入口

本地开发和答辩演示入口：

```text
MES System/src/main/java/com/example/messystem/MesBackendApplication.java
```

服务器部署入口：

```text
MES System/run-backend.sh
```

标准 Jersey Web 入口：

```text
MES System/src/main/java/com/example/messystem/MesApplication.java
MES System/src/main/webapp/WEB-INF/web.xml
```

前端入口：

```text
MES System/src/main/webapp/index.html
```

已删除旧的 `run-demo.*` 和 `com.example.messystem.demo.DemoServer`，避免不同入口暴露不同接口。
