# 腾讯云轻量应用服务器部署步骤

服务器公网 IP：`119.45.196.92`

本项目现在仿照 `PathFy-Uni` 的部署模式：

```text
浏览器 -> Nginx:80
静态页面 -> MES System/src/main/webapp
/api/* -> 反向代理到 127.0.0.1:8080
后端入口 -> com.example.messystem.demo.DemoServer
进程守护 -> Supervisor
```

也就是说，公网只访问：

```text
http://119.45.196.92/
```

后端 `8080` 端口只在服务器本机监听，不需要对公网开放。

## 1. 腾讯云控制台放通端口

进入轻量应用服务器控制台，找到实例的防火墙规则，添加入站规则：

```text
协议：TCP
端口：80
来源：0.0.0.0/0
说明：MES Web
```

如果之后配置 HTTPS，再额外开放：

```text
协议：TCP
端口：443
来源：0.0.0.0/0
说明：MES HTTPS
```

不建议开放 `8080`。本项目的 `8080` 只给 Nginx 在服务器内部访问。

## 2. SSH 登录服务器

在本机 PowerShell 执行：

```powershell
ssh root@119.45.196.92
```

如果你的服务器默认用户不是 `root`，可能是：

```powershell
ssh ubuntu@119.45.196.92
```

## 3. 安装运行环境

以下命令适用于 Ubuntu/Debian：

```bash
sudo apt update
sudo apt install -y openjdk-21-jdk maven git nginx supervisor
java -version
mvn -version
nginx -v
supervisord -v
```

必须确认 `java -version` 显示的是 JDK 21。不要改成其他 JDK 版本。

## 4. 拉取项目代码

推荐把仓库放在 `/www/wwwroot/mes`：

```bash
sudo mkdir -p /www/wwwroot
sudo mkdir -p /www/wwwlogs
cd /www/wwwroot
sudo git clone <你的远程仓库地址> mes
sudo chown -R root:root /www/wwwroot/mes
```

本仓库里的 Java 项目目录名是 `MES System`，路径中有空格。为了让 Nginx 和 Supervisor 配置更稳定，创建一个无空格的软链接：

```bash
sudo ln -sfn "/www/wwwroot/mes/MES System" /www/wwwroot/mes-app
```

后续部署统一使用：

```text
/www/wwwroot/mes-app
```

## 5. 配置数据库环境变量

如果当前只是课堂展示内存版流程，可以先不配置数据库。

如果要连接阿里云 RDS PostgreSQL，在服务器项目根目录创建 `.env`：

```bash
cd /www/wwwroot/mes
sudo nano .env
```

填写真实数据库信息：

```env
MES_DB_HOST=你的RDS PostgreSQL地址
MES_DB_PORT=5432
MES_DB_NAME=MESSystem
MES_DB_USER=MESSystem
MES_DB_PASSWORD=你的数据库密码
```

注意：`.env` 不要提交到 Git。仓库里只保留 `.env.example` 或 `deploy/env.production.example`。

## 6. 编译项目并本机测试后端

```bash
cd /www/wwwroot/mes-app
chmod +x run-backend.sh
MES_HOST=127.0.0.1 MES_PORT=8080 MES_OPEN_BROWSER=false ./run-backend.sh
```

看到类似输出说明后端启动成功：

```text
MES demo server started: http://127.0.0.1:8080/
Listening on 127.0.0.1:8080
```

另开一个 SSH 窗口测试：

```bash
curl http://127.0.0.1:8080/api/orders
```

测试完成后，在运行后端的窗口按 `Ctrl+C` 停止。

## 7. 配置 Supervisor 守护后端

项目已经提供配置模板：

```text
deploy/supervisor/mes-backend.conf
```

复制到 Supervisor 配置目录：

```bash
sudo cp /www/wwwroot/mes/deploy/supervisor/mes-backend.conf /etc/supervisor/conf.d/mes-backend.conf
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl status
```

如果状态不是 `RUNNING`，查看日志：

```bash
sudo tail -n 100 /www/wwwlogs/mes-backend-error.log
sudo tail -n 100 /www/wwwlogs/mes-backend-access.log
```

常用命令：

```bash
sudo supervisorctl restart mes-backend
sudo supervisorctl stop mes-backend
sudo supervisorctl start mes-backend
```

## 8. 配置 Nginx 作为公网入口

项目已经提供配置模板：

```text
deploy/nginx/mes.conf
```

复制到 Nginx 站点配置目录：

```bash
sudo cp /www/wwwroot/mes/deploy/nginx/mes.conf /etc/nginx/sites-available/mes.conf
sudo ln -sfn /etc/nginx/sites-available/mes.conf /etc/nginx/sites-enabled/mes.conf
```

如果默认站点占用了 80 端口首页，可以移除默认站点链接：

```bash
sudo rm -f /etc/nginx/sites-enabled/default
```

检查并重载 Nginx：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 9. 验证部署结果

服务器内部验证：

```bash
curl http://127.0.0.1:8080/api/orders
curl http://127.0.0.1/api/orders
```

本机浏览器访问：

```text
http://119.45.196.92/
```

如果网页能打开，且页面中的订单、工单、仓储等按钮能加载数据，就说明前后端部署链路成功。

## 10. 后续更新部署

每次组员推送新代码后，在服务器执行：

```bash
cd /www/wwwroot/mes
sudo git pull
sudo ln -sfn "/www/wwwroot/mes/MES System" /www/wwwroot/mes-app
cd /www/wwwroot/mes-app
sudo chmod +x run-backend.sh
sudo supervisorctl restart mes-backend
sudo nginx -t
sudo systemctl reload nginx
```

如果只是改了前端静态页面，通常重载 Nginx 即可；如果改了 Java 后端代码，要重启 `mes-backend`。

## 12. 常见问题

如果浏览器无法访问网页，先检查腾讯云防火墙是否开放 `80`，再检查 Nginx：

```bash
sudo nginx -t
sudo systemctl status nginx
```

如果网页能打开但接口报错，检查后端：

```bash
sudo supervisorctl status
curl http://127.0.0.1:8080/api/orders
sudo tail -n 100 /www/wwwlogs/mes-backend-error.log
```

如果 `8080` 被占用：

```bash
ss -lntp | grep 8080
```

可以修改：

```text
deploy/supervisor/mes-backend.conf
deploy/nginx/mes.conf
```

把两个文件里的 `8080` 同时改成新的端口，然后重新复制配置、重启 Supervisor、重载 Nginx。
